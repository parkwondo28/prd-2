## 1. Overview

### Problem Statement

- **문제 정의**: 1인 가구 작업자(프리랜서, 직장인 등)들이 집 근처에서 일할 공간을 찾을 때, 정보 부족으로 인해 여러 카페를 전전하며 시간과 에너지를 낭비하고 있습니다.
- **왜 중요한가**: 기존 지도 앱이나 SNS는 '인테리어'와 '음식' 위주의 정보를 제공할 뿐, 작업자에게 필수적인 '업무 환경(콘센트, 와이파이, 소음)' 정보는 제공하지 않습니다.
- **타겟 고객**: 서울 및 수도권 거주 30~50대 프리랜서, 재택근무자, 취업 준비생.
- **전략적 적합성**: '카공족'과 '카페 점주' 사이의 갈등을 중재하고, 방치된 유휴 공간을 효율적인 작업 공간으로 탈바꿈시킵니다.
- **핵심 전제/가정**: 사용자들은 작업에 최적화된 정보를 얻을 수 있다면 기꺼이 이용 에티켓을 준수하고 커뮤니티에 참여할 것이다.

### Proposed Work

- **솔루션 개요**: 동네 지도를 기반으로 콘센트, 와이파이, 소음 등 **'작업 환경 데이터'**에 특화된 탐색 서비스를 제공합니다.
- **핵심 기능**:

1. **작업 환경 상세 필터**: 콘센트 위치, 와이파이 속도, 배경음악 크기별 맞춤 검색.
2. **작업자 전용 한 줄 리뷰**: "책상이 넓음", "조명이 밝음" 등 실제 작업자들의 생생한 후기.
3. **에티켓 캠페인**: 카페 점주와 상생하기 위한 '이용 수칙' 가이드라인 제공.

- **사용자 플로우**: 앱 실행 → 위치 기반 지도에서 카페 탐색 → '콘센트/와이파이' 필터 적용 → 작업자 리뷰 확인 후 이동 → 이용 에티켓 확인 및 매장 이용 → 정보 업데이트 및 리뷰 작성.

### 기술 스택

- **지도 라이브러리**: react-leaflet (v5.0.0) + OpenStreetMap
- **지도 엔진**: Leaflet (v1.9.4)
- **지도 타일**: OpenStreetMap 타일 (https://{s}.tile.openstreetmap.org/{z}/{x}/{y}.png)
- **선택 이유**:
  - API 키 불필요 (완전 무료)
  - 한국 지도 타일 지원
  - React와 통합 용이
  - 커스터마이징 가능
  - 모바일/데스크톱 모두 지원

## 2. Decision / Risk

### Stakeholders

- **직접 이해관계자**: 서비스 운영팀, 1인 작업자, 개인 카페 점주.
- **간접 이해관계자**: 지도 데이터 제공사, 지역구 상권 활성화 부서.

### Risk

| 주요 위험 요소              | 발생 가능성 | 영향도 | 대응 방안                                                             |
| --------------------------- | ----------- | ------ | --------------------------------------------------------------------- |
| **점주와의 갈등**           | 높음        | 높음   | '카공족'이 아닌 '매너 있는 작업자' 문화 조성, 한가한 시간대 홍보 협의 |
| **정보 부정확성**           | 높음        | 중간   | 사용자 제보 시스템(정보 수정 요청) 활성화 및 참여 보상 지급           |
| **데이터 수집 한계**        | 중간        | 중간   | 초기 서포터즈 운영을 통한 핀포인트 지역(마포구 등) 데이터 집중 확보   |
| **OpenStreetMap 타일 제한** | 낮음        | 낮음   | 타일 사용량이 많을 경우 대체 타일 제공자 사용 (Mapbox 등)             |

### Dependencies

- **외부 서비스**: OpenStreetMap (지도 타일), 카카오톡 공유 (정보 공유).
- **필요 데이터**: 지역별 카페 기본 정보(상호명, 주소), 좌석별 콘센트 유무 정보.
- **필요 인력**: 초기 현장 실사 및 데이터 검수 인력, 사용자 문의 대응 운영자.
- **기술 의존성**: react-leaflet, leaflet, @types/leaflet

## 3. Success Criteria

### 핵심 지표 (2-3개)

- **지표명**: 정보 상세 보기 대비 지도 연결(방문 의사) 비율
- **측정 방법**: 특정 카페의 상세 페이지를 본 후 '길찾기'를 누른 비율
- **목표 수치**: 15% 이상

- **지표명**: 시범 지역 데이터 확보 수
- **측정 방법**: 마포구 등 특정 구역 내 '일하기 좋은 카페' 등록 수
- **목표 수치**: 50곳 이상 (3개월 내)

- **지표명**: 실제 이용자 수
- **목표 수치**: 가입자 500명 달성 및 유지

### 보조 지표

- 앱 내 '에티켓 가이드' 확인 횟수, 리뷰 작성률(방문자 대비 리뷰 작성 비율).

## 4. Definition of Done

### 필수 조건 (Done)

- 지도 위 카페 표시 및 '작업 환경 필터' 3종(콘센트, 와이파이, 소음) 정상 작동.
- 사용자가 직접 정보를 수정하거나 리뷰를 남길 수 있는 기능.
- 카페 상생을 위한 '이용 에티켓' 화면 구성.
- react-leaflet 기반 지도가 정상적으로 로드되고 마커가 표시됨.

### 개선 필요 (but 출시 가능)

- 카페별 메뉴판 사진 및 가격 정보 (초기에는 외부 링크로 대체 가능).
- 카페 사장님이 직접 소식을 올리는 공지사항 기능.

### 치명적 (출시 불가)

- 와이파이 유무 정보가 실제와 50% 이상 다를 경우.
- 앱 실행 시 지도가 뜨지 않거나 로딩이 10초 이상 걸리는 경우.
- 지도 타일이 로드되지 않거나 마커가 표시되지 않는 경우.

## 5. Nope (하지 않을 것)

- **기능명**: 실시간 좌석 예약 시스템
- **이유**: 점주들의 시스템 연동 부담이 크고 관리가 어려움.
- **대안**: 사용자의 '현재 혼잡도' 리뷰 정보로 대체.
- **재고려 시점**: 점주용 관리 페이지 보급이 완료된 시점.

- **기능명**: 전국의 모든 카페 등록
- **이유**: 데이터 신뢰도를 위해 '작업하기 좋은 곳'만 엄선하기 위함.
- **대안**: 사용자 추천(제보)을 받은 곳 위주로 순차적 업데이트.
- **재고려 시점**: 시범 서비스 안정화 이후 지역 확장 시.

- **기능명**: 유료 지도 API 사용
- **이유**: 비용 절감 및 오픈소스 생태계 활용.
- **대안**: OpenStreetMap 기반 무료 지도 사용.
- **재고려 시점**: 서비스 규모가 커져서 고급 지도 기능이 필요한 시점.

## 6. Post-Launch Plan

### v1.1 (Quick Wins)

- **리뷰 보상 체계**: 유용한 정보를 공유한 사용자에게 작은 혜택(배지 등) 제공.
- **사장님 한마디**: 점주가 "우리 카페는 노트북 사용을 환영합니다"라고 표시할 수 있는 기능.

### v2 (Major Features)

- **시간대별 분위기 통계**: 요일/시간별 소음 및 혼잡도 리포트 제공.
- **작업자 커뮤니티**: 같은 카페에서 일하는 사람들끼리 소통하거나 정보를 나누는 게시판.

### v3+ (Vision)

- **워크스팟 멤버십**: 전국 제휴 카페에서 할인이나 혜택을 받는 '디지털 노마드 패스' 출시.
- **공간 큐레이션**: 단순 카페를 넘어 공유 오피스, 도서관 등으로 검색 범위 확대.
